# Тестовое задание для Компас Плюс

## Что внутри

Проект состоит из двух основных частей:

- `org.example.Server` — TCP-сервер, принимает XML-сообщения, проверяет текст на запрещенные слова и отправляет статус.
- `org.example.Client` — консольный клиент, отправляет сообщения на сервер.

Дополнительно:

- SQLite-хранилище через `MessageRepository` / `SqliteMessageRepository`
- интеграционные тесты полного сценария клиент-сервер
- настройки в `src/main/resources/application.properties`

## Требования

- JDK 21+
- Gradle Wrapper (уже в репозитории)

## Настройки

Основные параметры лежат в `src/main/resources/application.properties`:

- `app.host` — хост сервера
- `app.port` — порт
- `app.datetime.format` — формат времени в сообщении
- `app.db.url` — URL SQLite (по умолчанию файл в корне проекта)
- `app.forbidden.words.path` — путь к файлу с запрещенными словами

## Как запустить

### Вариант 1 — через IDE

1. Запустить `org.example.Server` (метод `main`).
2. Запустить `org.example.Client` (метод `main`).
3. В клиенте использовать команды:
   - `-m <текст>` — отправить сообщение
   - `-h` — выйти

### Вариант 2 — запуск через консоль

Откройте два терминала в корне проекта.

В первом запустите сервер:

```bash
./gradlew runServer
```

Во втором запустите клиент:

```bash
./gradlew runClient
```

Команды клиента:

- `-m <текст>` — отправить сообщение
- `-h` — выйти

### Проверка только тестами

```bash
./gradlew test
```

## Тестирование

Основная проверка сценариев делается автоматически тестами, ручной запуск не обязателен.

Что покрыто:

- корректный статус для обычного сообщения
- корректный статус для сообщения с запрещенным словом
- обработка длинного сообщения (больше типичного TCP-сегмента)

То есть приложение можно валидировать полностью через тесты:

```bash
./gradlew test
```
